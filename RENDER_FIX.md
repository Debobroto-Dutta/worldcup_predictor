# Render Database Fix Guide

## Problem
The error `sqlite3.OperationalError: no such table: user` occurs because the database tables haven't been created on Render.

## Immediate Fix (For Current Deployment)

### Option 1: Run Database Initialization via Render Shell
1. Go to your Render dashboard
2. Navigate to your web service
3. Click on "Shell" tab
4. Run the following command:
   ```bash
   python init_render_db.py
   ```
5. Wait for the success message
6. Your database is now initialized!

### Option 2: Trigger a Redeploy
1. The `render.yaml` has been updated to automatically run `init_render_db.py` during build
2. Go to your Render dashboard
3. Click "Manual Deploy" → "Clear build cache & deploy"
4. This will rebuild and initialize the database automatically

## What Was Fixed

### Updated render.yaml
Changed the build command from:
```yaml
buildCommand: pip install -r requirements.txt
```

To:
```yaml
buildCommand: pip install -r requirements.txt && python init_render_db.py
```

This ensures the database is initialized automatically on every deployment.

## Verification Steps

After running the fix, verify the database is working:

1. Try to register a new user at: `https://your-app.onrender.com/`
2. Check the logs in Render dashboard for any errors
3. The registration should now work without the "no such table: user" error

## Understanding the Issue

### Why This Happened
1. Render creates an empty database when you deploy
2. The application code doesn't automatically create tables
3. Tables must be explicitly created using `db.create_all()`

### The Solution
The `init_render_db.py` script:
- Creates all database tables (user, match, prediction)
- Creates a default admin user
- Runs automatically during the build process

## Future Deployments

With the updated `render.yaml`, future deployments will:
1. Install dependencies
2. Automatically initialize the database
3. Start the application

The database initialization is idempotent - it won't recreate tables if they already exist.

## Troubleshooting

### If you still see the error after fix:
1. Check Render logs for build errors
2. Verify DATABASE_URL environment variable is set
3. Try clearing build cache and redeploying
4. Run `python init_render_db.py` manually via Shell

### If admin user already exists:
The script checks for existing admin user and won't create duplicates. You'll see:
```
✓ Admin user already exists
```

### Database Connection Issues:
If you see connection errors, verify:
- Database service is running in Render
- DATABASE_URL environment variable is correctly set
- Database and web service are in the same region

## Additional Notes

### SQLite vs PostgreSQL
- The error shows SQLite, but Render uses PostgreSQL
- The code works with both (SQLAlchemy handles the differences)
- Make sure DATABASE_URL points to your Render PostgreSQL database

### Environment Variables
Ensure these are set in Render:
- `DATABASE_URL` - Auto-set by Render when you link the database
- `SECRET_KEY` - Auto-generated by Render
- Email settings (optional, for password reset feature)
